## Proof pushed on-chain ‚Äî Sui devnet (publicly verifiable)
The dregg proof is not just enforced in our program logic ‚Äî the real BN254 Groth16 proof
(`perloan-prep/proof_A.json`, the STARK‚Üíapex‚ÜíGroth16 wrap) is **verified natively on Sui**
via `sui::groth16`. Package `dregg_verifier` @ `0x1ef488bbf6afe48c2e0236075c94a0dc6b5b017946bdade77f99f1a23bcf0478`.

| Case | Input | Result | Tx digest |
|------|-------|--------|-----------|
| Valid proof | real `proof_A` (vk+publics+proof) | ‚úÖ **success**, emits `Verified{ok:true}` | `DaATa2B2iDdPupHKioKcUwBPBHmJ1nSFYdDniVVXTnZS` |
| Tampered proof | same, 1 byte flipped (`d4`‚Üí`d5`) | üõ° **aborted** `EProofInvalid` (57005 / 0xDEAD) | `6bYtRnjN7gf8bVbffjQUTaK7EdyHWTLrjNhmXtBeadSL` |

`assert_verify` aborts unless `sui::groth16::verify_groth16_proof` returns true, so a **successful
transaction *is* the on-chain proof of verification** ‚Äî and the negative control shows the chain
genuinely checks it (a single flipped byte is rejected). These are real Sui devnet transactions
(explorer-verifiable now; devnet state may be pruned on the next reset ‚Äî mirror to testnet/mainnet
for permanence). Verifier source: `onchain-sui/move/dregg_verifier/sources/verifier.move`.

## Full Sui-native app stack + pentest (2026-07-14)
The whole platform now runs Sui-native (contract ‚Üí SDK ‚Üí operator API ‚Üí web wallet). Contract v4
`dregg_lending_async` @ `0x329d60b50dbaee425c6c681bab5789dbb437ddfc5ab9600ab45c75298c1d4aa4`, with a
**non-custodial** borrow path: the borrower sends the tx + supplies collateral; the operator's dregg
authorization is an **ed25519 attestation over the exact loan terms, verified on-chain** via `sui::ed25519`
(`disburse_attested`) ‚Äî no OperatorCap two-party tx, no custody. `sui move test` 5/5.

**Sui penetration test ‚Äî 6/6 blocked** (live devnet, `app/sui-sdk/test/pentest-sui.mjs`):

| Attack | Result | Why |
|--------|--------|-----|
| P1 forged attestation | üõ° BLOCKED | `MoveAbort` EBadAttest ‚Äî ed25519 verify fails |
| P2 tampered terms (sign 500, submit 501) | üõ° BLOCKED | `MoveAbort` EBadAttest ‚Äî message binds every term |
| P5 repay another's position (seize collateral) | üõ° BLOCKED | `MoveAbort` ENotBorrower ‚Äî **found + fixed in this port** (repay returned collateral to caller) |
| P6 over-cap disburse (debt > pool cap) | üõ° BLOCKED | `MoveAbort` EOverCap |
| P3 junk-collateral borrow (unregistered type) | üõ° BLOCKED | operator registry 400 |
| P4 over-LTV borrow | üõ° BLOCKED | server max-check + dregg 403 |

P5 was a real vulnerability surfaced by building the Sui pentest ‚Äî the Move `repay` returned the
(over)collateral to whoever called it, so an attacker could pay a small debt and seize a larger
collateral. Fixed with a `pos.borrower == sender` check + a regression test (`repay_rejects_non_borrower`).

**Verified live via the SDK + operator API on devnet:** the operator API's ed25519 attestation is
accepted on-chain by `disburse_attested` (integration test `app/operator-api/test-borrow-sui.sh`), and the
full lending loop (init‚Üídeposit‚Üídisburse_attested‚Üírepay) runs through `@assay/sui-sdk`
(`app/sui-sdk/test/onchain-attested.mjs`). The whole stack is clickable locally: `bash
app/sui-harness/dev-up-sui.sh` ‚Üí `cd app/web && npm run dev` (Sui wallet via `@mysten/dapp-kit`,
`/faucet` for test coins).
